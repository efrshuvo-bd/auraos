# Documented serial transcript for AuraOS kernel bring-up on QEMU virt.
# `scripts/run-qemu.ps1` uses `-machine virt,gic-version=2`, raw `-kernel build/aura-kernel.bin`,
# `-initrd build/initrd.cpio`, and muxes PL011 UART + VirtIO console onto stdio
# (order may interleave slightly).
#
# GUI path (`scripts/run-qemu-gui.ps1`): same serial mux, plus `-device ramfb` and
# a host display window (ramfb is the visible surface by default).
# Success = serial smoke lines AND a visible 480×800 teal paint in the QEMU window
# (not "Guest has not initialized the display (yet)"). Kernel activates ramfb with
# a fw_cfg DMA write of RAMFBCfg; DATA-register stores alone leave the placeholder.
# On Windows the script prefers `-display gtk` (Scoop SDL often hangs before serial).
# GTK pixbuf/Adwaita warnings are cosmetic host packaging noise.
# Override: `-DisplayBackend sdl|gtk|default`.
# Pass `-VirtioGpu` to also add `-device virtio-gpu-device,bus=virtio-mmio-bus.1`
# for SCRUM-42 scanout (resource create + SET_SCANOUT + flush). Default GUI boots
# still use ramfb as the visible surface; with `-VirtioGpu` expect scanout serial
# proof (and a checkerboard frame when the GPU owns the window).
# Default GUI boots expect `display: no virtio-gpu device` then
# `display: ramfb mapped… (fw_cfg DMA)` / `display: ramfb smoke ok…`; headless skips ramfb.
#
# Kernel early boot / sched lines → UART
# Guest init/agent/shell writes     → VirtIO console TX (SYS_WRITE)
# Guest SYS_READ                   → VirtIO console RX (polled + GIC IRQ drain; empty unless typed)
# VirtIO-blk                       → build/ab-slots.img on virtio-mmio-bus.2 (prepare-ab-disk.ps1)
#
# Init yields until Agent Core posts READY; shell presents Home+Agent UI, then
# requests help + system_status over IPC from that UI path. Shell asks agent to
# shutdown so the scheduler can reach idle. During shell's busy-wait, CNTP may
# preempt several times; up to 3 `sched: preempt pid=…` lines are logged.
